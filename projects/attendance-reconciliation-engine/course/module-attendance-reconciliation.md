# Module 4: Attendance Reconciliation

## Learning Objectives

- Map planned and actual attendance inputs.
- Write deterministic exception rules.
- Create a human review queue.
- Preserve audit evidence.

## Worked Example

Worker 002 is planned for 08:00 to 16:00, clocks in at 08:22 and clocks out at 15:30. With a five-minute grace window the system classifies late arrival and early departure.

## Validation Checklist

- Inputs are synthetic or authorised.
- Worker identifiers are neutral.
- Rule thresholds are configurable.
- Exceptions include source references.
- Reports disclose limitations.
