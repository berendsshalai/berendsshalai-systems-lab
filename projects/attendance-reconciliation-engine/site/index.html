For Non-Technical Reviewers

This project helps explain a practical operations problem: people are scheduled for shifts, people clock in and out, and someone has to find what does not line up. The engine does not calculate payroll. It creates a structured review list so a human can check exceptions with evidence.

01

Rostered but absent

A worker had a planned shift but no matching clock event was found.

02

Present but unrostered

A clock event exists where there was no planned shift for that worker, site, and date.

03

Late or early

Arrival and departure differences are measured against configurable grace windows.

04

Needs review

Unmatched or incomplete clock records are surfaced instead of being hidden.

05

Coverage gap

A planned work window needs more people than the roster provides.

06

Roster overlap

The same worker is planned in two places at the same time.

Try the Local Workflow

python -m pip install -e .
python -m attendance_reconciliation.synthetic data/demo
python -m attendance_reconciliation.cli reconcile --shifts data/demo/shifts.csv --attendance data/demo/attendance.csv --output out/report.json
python -m attendance_reconciliation.cli review-roster --shifts data/demo/shifts.csv --coverage data/demo/coverage.csv --output out/roster-review.json
python -m pytest tests

The sample files contain fictional workers and fictional sites only. They are designed to explain the workflow without exposing private operations, API tokens, or actual financial data.

This project is an independent clean-room implementation based on a common operational problem. It contains no private organisational source code, data, branding, API tokens, or actual financial records.
