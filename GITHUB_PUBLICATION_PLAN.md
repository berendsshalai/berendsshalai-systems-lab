# GitHub Publication Plan

GitHub is the primary public operating surface for the Systems Lab. Wix is reserved for later portfolio presentation once repository evidence is stable.

## Publication Order

1. Keep work in clean local directories.
2. Run tests and privacy gate.
3. Initialise local Git history from the clean repository only.
4. Inspect staged diff and publication report.
5. Create public GitHub repositories under `berendsshalai`.
6. Push sanitised history.
7. Enable GitHub security features.
8. Create versioned releases.
9. Deploy GitHub Pages demos.
10. Only then update the personal website and Wix portfolio summaries.
