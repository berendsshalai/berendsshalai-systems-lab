# Ecosystem Roadmap

## Phase 0: Safety and Inventory

- Establish clean-room policy.
- Create private sanitisation area excluded from Git.
- Inventory generic system categories without private names.
- Define publication gates.

## Phase 1: Shared Foundation

- Build repository standards and reusable templates.
- Implement privacy gate and synthetic data strategy.
- Create project registry, course framework, MCP standard and SEO map.

## Phase 2: Flagship Project

- Complete Attendance Reconciliation Engine before expanding optional features.
- Stabilise CLI, browser demo, MCP server, tests, docs and course module.

## Phase 3: Workforce Systems

- Build roster optimisation, payroll-hours audit and recruitment routing.

## Phase 4: Financial and Document Systems

- Build revenue reconciliation, document capture and hourly sales processing.

## Phase 5: Operations Platform

- Build evidence monitoring, dashboard and shared workflow communication.

## Phase 6: Public Education

- Publish Practical Operations Automation course content from completed project evidence.

## Phase 7: Release and Portfolio

- Create public GitHub repositories only after quality, privacy and manual publication review pass.
- Update website and Wix portfolio after GitHub artefacts are stable.
