# Clean-Room Policy

This ecosystem reconstructs public systems from generic functional requirements, open standards, synthetic data and independently written code.

It must not copy or reveal private source code, employer-owned macros, internal workbooks, production formulas, private reports, screenshots, credentials, internal process manuals, customer information, employee information, private file paths, private names or proprietary prompts.

Every public project must include:

- `CLEAN_ROOM_DECLARATION.md`
- `NON_AFFILIATION.md`
- `DATA_PROVENANCE.md`
- `PRIVACY.md`
- `SECURITY.md`
- `THREAT_MODEL.md`

The standard disclosure is:

> This project is an independent clean-room implementation based on a common operational problem. It contains no private organisational source code, data, branding or proprietary documentation.
